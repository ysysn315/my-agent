#### OpsRAG 智能检索与运维问答平台
**项目概述：**构构建以 RAG 检索增强 为核心的运维问答 Agent，融合 **知识检索（RAG）+ 对话编排 + AIOps 运维闭环**，实现问题自动应答与故障智能排查，降低值班成本并提升团队效率。
**技术栈：**Python / FastAPI / LangChain / LangGraph / DashScope/ Milvus/ Docker/Redis
**核心工作：**
- **Agent 架构与状态管理：** 基于 **LangGraph+LangChain **构建分层级智能体体系，设计** **Chat ReAct 与 Ops Task 双独立链路，引入 Redis 实现会话状态存储，解决多轮对话中的上下文丢失问题，结合 SSE 流式输出优化交互体验。

- **RAG 方案设计与调参：** 搭建基于** Milvus** 的向量检索引擎，实现BM25 关键词 + 向量语义**混合检索**；引入BGE-Reranker对 Top-K 结果进行重排；实现查询扩展与查询改写模块，补全用户模糊提问上下文。

- **RAG评测与优化闭环：**构建离线评测与回归机制，围绕检索与生成质量设计 Hit@3 / MRR / NDCG / SourceHit 等核心指标并持续对比实验，结合误检样本迭代召回与重排策略；相比 baseline，MRR 0.6708→0.7508、NDCG@3 0.6776→0.7506，稳定提升复杂场景下的检索准确性与回答可靠性。

- **AIOps 闭环落地：** 基于 LangGraph 构建 Supervisor-Worker 拓扑结构，设计 Planner、Operation 与 Reflection 的状态流转机制；通过 **LangChain Tools** 标准化接入 Prometheus 监控、日志查询、时间查询及知识库，将告警分析与建议生成转化为可复用的自动化工作流。

#### **Micro-GPT-Core：轻量级 Decoder-only 架构复现与**全链路训练**实践 (25.8M)**
**技术栈：PyTorch, Transformer,MoE, DPO, GRPO，LoRA, RoPE, KV Cache**
**核心工作：**

  - 底层实现：基于 PyTorch 手写 Transformer Decoder和**MoE**混合专家架构，手动实现** RoPE** 旋转位置编码与** RMSNorm** 算子以提升训练稳定性；引入** GQA** (分组查询注意力) 与 **KV Cache** 机制，显著降低推理显存占用与延迟。


- 训练闭环：打通 Pretrain → SFT →RL 全流程；应用 **LoRA** 实现高效指令微调，并集成 **DPO** (RLHF) 与 **PPO/GRPO** (RLAIF) 算法，验证了从离线偏好学习到在线 AI 反馈的完整对齐路径。


- 性能优化与评估：应用 AMP 混合精度与梯度累积技术。使用评估脚本分析模型在指令遵循与推理速度上的性能差异。
