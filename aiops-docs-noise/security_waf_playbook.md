# WAF 安全策略手册

目标：
- 防御 SQL 注入
- 防御 XSS
- 防御 CC 攻击

告警分级：
- P1：业务不可用
- P2：可疑攻击上升
- P3：单点异常

本手册面向安全运营，与 JVM OOM、GC 压力或数据库慢查询无直接关系。

## 策略编排

### 基础防护
- 开启常见 Web 漏洞规则集
- 开启请求体与 URL 参数关键字检测
- 对异常 UA、Referer 建立拦截策略

### 速率限制
- 按 IP 进行每分钟请求频次限制
- 对登录、下单、支付回调等接口单独限流
- 对突发峰值设置弹性阈值与自动封禁

### 地域与访问控制
- 仅开放业务覆盖地域
- 对高风险 ASN 段设置挑战模式
- 对管理后台路径增加白名单策略

## 告警与响应流程

1. 收到攻击告警后确认影响范围
2. 根据攻击类型应用对应防护模板
3. 跟踪误拦截比例并快速回调策略
4. 输出处置报告，沉淀规则库

## 常见误区

- 仅依赖默认规则，不做业务化策略
- 告警过多但未设置聚合与抑制
- 变更策略后未做回放验证

注：本文关注安全防护策略，和应用性能抖动、JVM 内存问题、数据库慢查询排障无直接关系。
