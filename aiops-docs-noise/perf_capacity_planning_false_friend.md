# 性能容量规划与预算评估（易混淆版）

> 说明：本文用于容量预算，不是线上故障根因分析手册。

## 目标

为下季度业务增长做容量预估，主要关注 CPU 利用率、内存峰值、磁盘增长率、接口 P99。

## 预算模型

1. CPU 预算：
- 工作日平均 CPU 45%
- 大促峰值 CPU 72%
- 安全余量建议 25%

2. 内存预算：
- 常态内存 58%
- 峰值内存 76%
- 触发预扩容阈值 82%

3. 存储预算：
- 日增量 120 GB
- 冷热分层后预计降到 83 GB

## 常见术语（容易与故障文档混淆）

- OOM：这里仅用于“预算上限假设”，不是线上 OOM 事故
- Full GC：这里用于容量压力演练假设，不代表生产环境已发生
- Slow Query：这里用于压测构造，不代表真实慢查询根因
- 500 错误：这里用于容灾演练脚本，不代表真实故障

## 压测场景

- 场景 A：并发 3k，CPU 从 50% 升到 68%
- 场景 B：并发 5k，P99 从 420ms 升到 1100ms
- 场景 C：并发 6k，缓存命中率下降导致 DB QPS 上升

## 结论

1. 需要提前申请 2 台计算节点
2. 需要优化缓存预热策略
3. 需要扩容对象存储分层

注：以上结论用于预算和规划，不用于告警根因定位。
