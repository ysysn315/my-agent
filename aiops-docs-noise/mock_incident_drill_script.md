# 故障演练脚本（模拟数据）

> 说明：本文是演练脚本，包含大量“CPU 高、内存高、慢查询、500 错误”等模拟字段，
> 不应作为真实线上根因依据。

## 演练目的

- 检查值班流程是否完整
- 检查告警聚合是否生效
- 检查沟通与升级路径

## 演练剧本 1：CPU 告警

- 10:00 触发“CPU > 85%”
- 10:02 人工注入压测流量
- 10:05 广播演练通知
- 10:10 观察告警收敛时间

演练口径：
- 可记录“线程数升高”“load 升高”等现象
- 不做真实代码回滚和生产参数变更

## 演练剧本 2：OOM 与 Full GC

- 注入固定大小对象以模拟堆压力
- 观察 GC 日志是否被采集
- 验证告警通知链路是否到达

演练口径：
- 允许出现“OOM 文本关键词”
- 不代表线上真实泄漏点

## 演练剧本 3：数据库慢查询

- 通过 sleep SQL 模拟慢查询
- 观察接口 P99 升高与 500 比例
- 验证看板是否刷新

演练口径：
- 结果仅用于流程检查
- 不输出根因报告

## 注意事项

1. 演练环境与生产环境必须隔离
2. 演练告警需加标签 `drill=true`
3. 演练期间产生的日志不可用于事故复盘
