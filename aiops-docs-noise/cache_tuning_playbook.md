# 缓存参数调优手册（非根因文档）

## 场景

在高并发场景下，缓存命中率下降会导致下游数据库压力增加，进而可能出现延迟上升。

## 参数建议

- 本地缓存 TTL：30s ~ 120s
- 分布式缓存 TTL：300s ~ 1800s
- 热点 key 预热窗口：5min
- 过期抖动：±10%

## 常见现象（容易混淆）

- CPU 升高：可能来自序列化开销和回源风暴
- 内存升高：可能来自大对象缓存和碎片
- 慢响应：可能来自缓存穿透导致的数据库回源
- 500 错误：可能来自下游超时后的熔断回退

## 处理动作（仅限缓存层）

1. 提升热点 key 命中率
2. 增加批量回填和异步刷新
3. 优化序列化格式
4. 增加本地缓存兜底

## 不在本文范围内

- JVM 内存泄漏定位
- 数据库索引治理
- 服务不可用恢复编排

注：本文是缓存调优手册，不是线上故障根因分析报告。
